Html Elements

Html generally stand for Hyper Text Markup Language. Html describes the structure of web pages using markup. Html was invented by Tim Berners-Lee a
British computer scientist. Originally, Html was developed with the intent of defining the structure of documents like headings, paragraphs, lists, and so forth
to facilitate the sharing of scientific information between researchers. Now, Html is being widely used to format web pages with the help of different tags
available in Html languages.

The Html Elements are the basic building blocks of a web page. they define the structure and content of a web page by using tags to enclose different types of content,
like text, images, links. Html elements provide the foundation for creating web pages but don't inherently convey specific meaning or context to the content they contain.
It usually consist of a start tag and end tag, with the content inserted in between. The Html elements is everything from the start tag to the end tag. Examples of Html
elements includes; '/div>', '/span>','/p>','/h1>', ecetera. Additionally, Html elements can be nested (elements can contain elements). All Html documents consist of nested
Html elements. for instances; all the elements from the DOCTYPES, /body, and /html are all Html elements. Furthermore, Html elements with no content are called empty
elements. for example, /br>, is an empty element without a closing tag. the /br> defines a line break.

Semantic Html Elements

Semantic Html Elements are special tags in Html that that provide meaning and context to the content they contain. They help describe the structure of web documents in
a way that is easily understandable for both human beings and machines. Essentially, semantic Html elements makes your code more organized, readable, and meaningful.

Differences Between Html Elements And Semantic Html Elements

The differences between the Html elements and the semantic Html elements includes some of the following:
  1. Html elements are the basic building blocks of a web page. They provide no inherent meaning or context to the content. Semantic Html file on the otherhand,
    provide meaning and context to the content they enclose.
  2. Html elements define the structure and content of a web page. While Semantic Html elements better structure and organize the content.
  3. Html elements is often used for styling and scripting purposes while semantic Html convey meaning and intent to assistive tech.
  4. Semantic html elements can improve SEO (Search Engine Optimization)as search engines understand the meaning and context of the content better. Html elements
    have limited impact on SEO
  5. Examples of Html elements are; div, span, p, h1-h6 ecetera. Example of the semantic html elements include; header, footer, nav, article, section, main, fiqure, aside, ecetera.

In conclusion, while Html elements provide the basic structure of a web page, semantic Html enhance the meaning and context of the content, leading to better readability,
accessibility, SEO and styling.

Closing Tags and Self-Closing Tags in Html

Closing tags in Html are used to explicitly mark the end of an Html elements. They are written by enclosing the content of the element betweeen an opening tag and a corresponding
closing tag. This helps define the boundaries of the element's content within the Html document, making the structure clear and organized. For example, in <'p> paragraphs contain here.
<'/p> - which is the closing tag indicates the end of the paagraph element, ensuring proper rendering and interpretation by web browsers.

Self-Closing tags in Html are used to define elements that do not contain any content and do not require a seperate closing tag. They are written with a forward slash before the closing
angle bracket, immediately following the opening tag. Self-closing tags represent standalone elements and are typically used for elements like, images, line breaks, and input fields.
For example, <'img sar='image.jpg'/> is a self-closing tag that define an image element without any content, while <'br> represent a line breal element.

Differences Between Closing Tags and Self-Closing Tags in Html

  1. Closing tags are used to explicitly mark the end of an Html element by closing the content within the opening and closing tags like, the <'div> content <'/div>.On the other hand,
    self-closing tags are used to define elements that do not have any content nested within them and don't require a seperate closing tag. they end with a forward slash before the close
    angle bracket, like <'img src=image.jpg"/>.
  2. Closing tags can enclose other Html elements or plain text content within them. While self-closing tags do not enclose any content. They are standalone and represent single
    elements without content.
  3. Closing tags provide clarity about where an elements and ends, making the structure of the document clearer. Self-closing tags simplify markup, especially for elements
    without content, making it more concise.
  4. Closing tags is compatible with all Html versions and widely supported by browsers. Self-closing tags is introduced in Html as Xhtml syntax but adopted in Html5. Supported
    in modern browsers and Html5-compatible documents.
  5. Examples of closing tags are;
    <'div> content <'/div>
    <'p> paragraph <'/p>
    <'span> text <'/span> ecetera.
    EXamples of self-closing tags include the following;
    <'img src=image.jpg"/>
    <'br />
    <'input type="text"/>, ecetera.

In summary, closing tags are used to mark the beginning and end of elements with content, while self-closing tags represent standalone elements without content, providing a
concise and clear way to define certain types of elements in Html.

Inline Elements and Block Elements

Inline elements in Html are elements that don't start on a new line and only take up as much width as necessary. They typically flow within a line of text or other inline elements.
Put differently, Inline elements can appear within sentences and do not have to apear on a new line of their own. The <'br>, <'i>, <'u>, <'em>, <'strong>, <'sup>, <'big>, <'small>,
<'li>, <'ins>, <'del>, <'code>, <'cite>, <'dfn>, <'kbd> and <'var> elements are all inline elements.

Block elements in Html are elements that start on a new line and take up the full width available. They create a "block" of content, pushing other elements to start on a new line
before and after them. They all start on their own new line, and anything that follows them appears on its own new line. For example, <'p>, <'h1> - <'h6>, <'ul>, <'ol>, <'dl>, <'pre>,
<'hr>, />, <'blockquote> and <'address> elements are all block level elements.

Differences Between Inline Elements and Block Elements

  1. The inline elements flow alongside surrounding content while Block elements start on a new line.
  2. While the inline elements occupies only necessary width, the block elements on the other hand, occupies full available width.
  3. The inline elements can not have a specified width and height while the block elements can have width and height specified.
  4. In inline elements, the margine and paddlings are applied horizontally, affecting surrounding contents while in block elements, the margins and paddlings can be around.
